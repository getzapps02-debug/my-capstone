## 2026-06-15 - Shortfall Definition

- A **Shortfall** is a user-selected period during which an Account Balance falls below the user's Safety Threshold.
- A Shortfall does not require the Account Balance to become negative.
- The user retains control over the Safety Threshold and which qualifying period to investigate.
- The **Investigation Period** starts at the most recent point before the Shortfall when the Account Balance was at or above the Safety Threshold.
- The Investigation Period ends at the lowest Account Balance before it returns to or exceeds the Safety Threshold.
- If the Account Balance does not recover, the Investigation Period ends at the latest available Transaction.
- The user can manually shorten or extend the Investigation Period.

## 2026-06-15 - Contributor Contract

- A **Contributor** is one or more eligible Transactions or Obligations that reduced the available buffer during the Investigation Period.
- Contributors are ranked by:
  1. Total monetary impact.
  2. Deviation from the previous Comparable Period.
  3. Timing proximity to crossing the Safety Threshold.
- Every Contributor ranking must expose its calculation, comparison basis, and source evidence.
- Transfers and duplicate Transactions are excluded from Contributor calculations.
- Refunds reduce the monetary impact of the related Contributor.
- Contributors with equal ranking values receive the same rank.
- Contributor language describes observable contribution or association and never claims proven causation.
- The **Comparable Period** is the immediately preceding period with the same number of calendar days as the Investigation Period.
- The Comparable Period must contain sufficient Transaction data to support comparison.
- If the Comparable Period is unavailable or incomplete, Contributors are ranked using monetary impact and timing proximity only.
- When historical comparison is unavailable, the explanation must disclose that limitation.
- The user can select a different date range as the Comparable Period.

## 2026-06-15 - Account and Balance Scope

- The MVP can store multiple Accounts but supports one Account per Investigation.
- Account Balances and Contributor findings are not combined across Accounts.
- Transfers between known Accounts are linked and excluded from spending and Contributor calculations.
- The Account Balance uses imported running-balance values when available.
- If running balances are unavailable, the Account Balance is reconstructed from an Opening Balance and ordered Transactions.
- The product displays any reconciliation difference before allowing the user to rely on an Investigation.

## 2026-06-15 - CSV Import Contract

- Required CSV mappings: Transaction Date, Amount, Description, and Account.
- Optional CSV mappings: Running Balance, Category, Transaction Type, Merchant, and Transaction ID.
- The user previews and confirms column mappings before import.
- The importer supports common date formats and either signed amounts or separate debit and credit columns.
- Malformed rows are rejected with specific reasons while valid rows can continue importing.
- Transaction ID is the primary duplicate identity when present.
- Without a Transaction ID, duplicate identity uses Account, Transaction Date, Amount, and normalized Description.
- Renaming or reordering an already imported file does not create duplicate Transactions.
- The user can correct mappings and retry without duplicating previously accepted Transactions.
- Import success means at least 95% of valid rows are interpreted correctly, not merely accepted.

## 2026-06-15 - Corrections and Recalculation

- The user can edit a Transaction's Category, Merchant, Description, Transaction Date, Amount, Transaction Type, and transfer status.
- The user can exclude a Transaction from an Investigation without deleting it.
- The user can delete individual Transactions, undo an entire import, or clear all local data after explicit confirmation.
- Any evidence correction immediately recomputes Account Balances, charts, Contributors, rankings, and Transaction Replay.
- Imported facts, user corrections, and derived findings are visibly distinguishable.
- Findings based on changed evidence are invalidated and cannot remain displayed as current.

## 2026-06-15 - Minimum Evidence and Refusal Behavior

- An Investigation can rank Contributors only when it has:
  - One Account.
  - An Opening Balance or imported running Balance.
  - Complete Transaction Dates and Amounts for the Investigation Period.
  - A valid Safety Threshold.
  - A reconciled Balance or a reconciliation difference explicitly accepted by the user.
- If evidence is insufficient, the product does not rank Contributors.
- The insufficient-evidence state identifies every missing or inconsistent input.
- The user can correct the evidence and rerun the Investigation.
- The product never fills evidence gaps with guesses.

## 2026-06-15 - Obligations and Context

- An **Obligation** is a recurring or one-time required payment entered or confirmed by the user and optionally linked to a matching Transaction.
- **Context** is a user-provided annotation such as an event, person, place, mood, need-or-want classification, emergency, tag, or note.
- Unconfirmed Obligations and Context are visibly labeled as user-provided.
- Context does not alter Contributor ranking in the MVP; it provides supporting explanation only.
- An Obligation affects Contributor ranking only through the Amount of its linked Transaction.
- Missing Context never blocks an Investigation.

## 2026-06-15 - Transaction Replay

- Transaction Replay processes Transactions from the start to the end of the Investigation Period in chronological order.
- Same-day Transactions use source order when available, then stable import order.
- Replay synchronizes the Account Balance chart, Category trend, Transaction list, Contributor evidence, and graph evidence.
- Replay controls include play, pause, previous, next, restart, and speed adjustment.
- Selecting a chart point or Transaction moves Replay to that event.
- Replay highlights the point where the Account Balance crosses the Safety Threshold.
- Reduced-motion mode replaces animation with step-by-step updates.
- Finishing or exiting Replay restores the complete Investigation view.

## 2026-06-15 - MVP Environment and Data Boundaries

- Supported environment: Windows 11 with a current version of Chrome or Edge.
- Each Account uses one Currency; mixed-Currency imports are rejected.
- Currency values are rounded to two decimal places.
- Dates are interpreted in the user's local timezone; CSV dates without times use local calendar dates.
- Local data persists across page refresh, browser restart, and application restart.
- The MVP sends no telemetry or financial data outside the local environment.
- Each Account supports up to 10,000 Transactions.
- Thirty days of history is recommended but not required when the Minimum Evidence standard is met.
- Sample data remains separate from personal data and can be reset independently.
- The user can export Transactions and permanently delete all local data.

## 2026-06-15 - Accessibility and Completion Metric

- The full core journey is operable by keyboard.
- Charts provide text summaries and tabular alternatives.
- Meaning never depends on color, hover, or animation alone.
- Transaction Replay respects reduced-motion preferences.
- Interactive controls have accessible names and visible focus indicators.
- The MVP targets WCAG 2.2 AA for the core journey.
- The five-minute usability measure starts when sample data is loaded.
- The measure ends when the participant identifies the top Contributor, opens its supporting evidence, and correctly explains that it is a Contributor rather than a proven cause.
- All five formative participants use the same curated synthetic scenario.
- At least four of five participants must complete the task without moderator guidance.
- Any severe misleading interpretation requires revision and retesting.
